Chapter 1: What Is a Database?

Objective

Learning Objectives¶

By the end of this chapter, you should be able to:

  1. Describe what a database is.

  2. Explain why databases are useful in organizations.

  3. Distinguish between data, information, databases, and database management systems.

  4. Compare a spreadsheet-based approach with a database-based approach.

  5. Use a small SQLite database to store and retrieve data.

1.1 Why Study Databases?¶

Every organization depends on data. Colleges store student records. Hospitals track patients and appointments. Retail companies manage customers, products, and orders. Manufacturers track inventory, suppliers, machines, and production schedules.

A database is a structured collection of related data. Databases help organizations store, organize, search, update, protect, and share data.

Databases matter because they allow people to answer important questions, such as:

  • Which students are enrolled in a course?

  • Which customers placed orders last month?

  • Which products are low in inventory?

  • Which employees worked on a project?

  • Which machines need maintenance?

In this book, we will focus mostly on relational databases and SQL, which is the language commonly used to work with relational databases.

1.2 Data vs. Information¶

Although people often use the words interchangeably, data and information are not exactly the same.

Data are raw facts or values.

Examples:

StudentIDFirstNameLastNameMajor
1001MayaJohnsonIST
1002CarlosRiveraBusiness
1003AminaPatelEngineering

Information is data that has been organized or interpreted so that it is useful.

Examples:

  • There are 3 students in the sample.

  • One student is majoring in Business.

  • Student 1003 is Amina Patel.

A database helps turn raw data into useful information.

1.3 What Is a Database?¶

A database is an organized collection of related data.

The word organized is important. A random list of names is not usually enough. A useful database has structure, rules, and relationships.

For example, a college registration database might include:

  • Students

  • Courses

  • Instructors

  • Course sections

  • Enrollments

These are related. A student enrolls in a course section. A course section is taught by an instructor. A course belongs to a department.

A database allows these related pieces of data to be stored together in a structured way.

1.4 What Is a DBMS?¶

A Database Management System, or DBMS, is software used to create, manage, and interact with databases.

Examples of database management systems include:

  • SQLite

  • MySQL

  • PostgreSQL

  • Microsoft SQL Server

  • Oracle Database

The database is the stored data.
The DBMS is the software that manages the database.

A DBMS helps users:

  • Add new data

  • Update existing data

  • Delete data

  • Search data

  • Protect data

  • Control who can access data

  • Prevent inconsistent or invalid data

1.5 Databases vs. Spreadsheets¶

Spreadsheets are useful tools, especially for small tasks. However, as data becomes larger, more connected, or shared by many users, spreadsheets can become difficult to manage.

SpreadsheetDatabase
Good for small, simple datasetsGood for larger, structured datasets
Often edited manuallyOften managed using rules and queries
Easy to duplicate inconsistent dataDesigned to reduce unnecessary duplication
Limited control over relationshipsDesigned to model relationships
Not ideal for many users editing at onceBetter for shared organizational systems

For example, a spreadsheet might store customer and order data in one large table. A database would usually separate customers, orders, and products into different but related tables.

1.6 Tables, Rows, and Columns¶

Relational databases organize data into tables.

A table is made of:

  • Columns: categories or fields of data

  • Rows: individual records

  • Cells: individual values

Example table: Student

StudentIDFirstNameLastNameMajor
1001MayaJohnsonIST
1002CarlosRiveraBusiness
1003AminaPatelEngineering

In database terminology:

  • A table is often called a relation.

  • A row is often called a tuple.

  • A column is often called an attribute.

We will use both everyday language and formal database language throughout this book.

1.7 A First Look at SQL¶

SQL stands for Structured Query Language.

SQL is used to communicate with relational databases. SQL can be used to:

  • Create tables

  • Insert data

  • Retrieve data

  • Update data

  • Delete data

  • Control access to data

A simple SQL query looks like this:

SELECT *
FROM Student;

This means: show all columns and rows from the Student table.

1.8 Why Structure Matters¶

Imagine storing all college data in one spreadsheet:

StudentNameMajorCourseNameInstructorRoom
Maya JohnsonISTIntro to DatabasesDr. Lee202
Carlos RiveraBusinessIntro to DatabasesDr. Lee202
Amina PatelEngineeringWeb DevelopmentDr. Gomez118

This might work at first, but problems appear quickly:

  • The instructor’s name may be repeated many times.

  • A room change might require updates in many rows.

  • A student’s name could be typed differently in different places.

  • It becomes difficult to track which students, courses, instructors, and rooms are truly separate things.

A database helps solve these problems by organizing data into related tables.

1.9 Common Types of Databases¶

Although this book focuses on relational databases, it is useful to know that there are several types of databases.

TypeBasic IdeaExample Use
Relational databaseData stored in related tablesStudent records, sales systems
Document databaseData stored as flexible documentsWeb applications, product catalogs
Graph databaseData stored as nodes and relationshipsSocial networks, recommendation systems
Key-value databaseData stored as key-value pairsCaching, fast lookups
Object-oriented databaseData stored as objectsSpecialized software systems

Relational databases remain widely used because they are powerful, structured, and supported by SQL.

1.19 Chapter Summary¶

In this chapter, you learned that a database is an organized collection of related data. Databases are used across organizations to store, manage, and retrieve data efficiently.

You also learned that:

  • Data are raw facts.

  • Information is data made meaningful.

  • A DBMS manages databases.

  • Relational databases organize data into tables.

  • Tables contain rows and columns.

  • SQL is used to communicate with relational databases.

In the next chapter, we will begin looking more closely at how data is modeled before it becomes a database.
